What is PlayCanvas

A quick intro of PlayCanvas: In a nutshell, PlayCanvas is a WebGL, open source games engine with a cloud based development platform that can run directly in the browser. Within a few clicks and no installs, users can create, iterate and publish a game all directly from the browser. This is one of our showcase projects. Seemore that was originally developed with ARM and has been updated a few times to take advantage of new engine features. Some of the features shown here: HDR lightmaps Hardware supported texture compression Draw call batching Asynchronous streaming of assets to allow for fast startup
Here's a screenshot of the Scene Editor which will be familar to most people. I will be a giving a small demo at the end that goes through the workflow of creating a project and publishing so stay tuned for that.

The capabilities of browser actually snuck up on me despite working on Flash games with PlayFish at the time. I thought Flash was pretty safe as you effectively wrote one and it worked 'everywhere'.
I was used to seeing and expecting 2D games like this in the browser. 3D games were very rare and usually expiremental.
It wasn't until this presentation in 2016 by PlayCanvas that I actually sat up and took notice. Will did a WebVR demo where was able to create and publish a build in front of us that we could try almost immediately. The individual components weren't anything new but the combination and that the build was running 3D graphics on a mobile, was hugely impressive to me. It wasn't something I previous thought was possible and it was like being given a whole new toolset all at once.

Game Portals

Web game portals are still going but are adapting to the times. Miniclip still have 70 million users per month on their site and are now a mobile publisher for mobile titles. A lot of these portals have been around since the Flash days (even when I was at Uni). At the moment, ad revenue is still the prime source of income for portals. While there has been some movement on Web Monetisation (see Coil), that still feels a way off to being the standard. Some portals have not been as lucky. Kongregate have started to wind down their games services and have stopped accepting new games from third party developers.

Interactive and playable ads is something that is becoming more popular over the last few years. Companies are using ads as a way to engage the user and get some 'buy in'. Especially if there is some sort of prize at the end such as a voucher. This is an older project for Call of Duty and allows the user to configure their favourite setup. This could be extended to be shared amongst frieds/social media which is promotion for the game.
Playable ads (typically served in mobile games as a web view) are used to drive UA by giving players the ability to sample some gameplay. In these examples above, Azur Games got a 60% increase of installs per 1000 ad impression compared to video. MyTona and their hidden object game, got a 4.6 multiple increase of click through rate than video ads (show video from site). The web allows developers to close the gap between players seeing something they are interested in and being able to engage with it. This has become even more important as games are now competing against other streaming services like Netflix, YouTube, Spotify for user attention.
The major messenger platforms support HTML5 games within the app itself as web view. This gives the games immediate access to a social graph and it becomes much easier for the player to share scores, take turns between friends. KaiOS

One example is Diablo running in the browser. It may be the first version, but it's Diablo in a browser at full speed which is pretty sweet. Other demos are retro emulators and even Godot.
WebXR is a draft for the new API to provide VR and AR experiences from the browser. This is an example of a PlayCanva user making a VR game for a Game Jam.
The AR side of things is more exciting for me and I'm expecting it be a game changer. On Android, it uses Ar Core under the hood and the fact that it runs from the browser, reduces the friction to needing an external app to try something that you see on a poster or find on a website. I can imagine browsing the IKEA store site to see if a chair fits in the space without the friction of installing their app. I really hope that iOS do implement this with Ar Kit.
